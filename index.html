Project 1: Cinema Movie Database

An interactable Cinema Movie Database created in both Android Java and Swift

Allows for the searching of movies by title, director, cast, and release date. Includes an option to filter by upcoming, and released movies. Once selected you can view the cinemas the movie is available at, and can be automatically searched on Google Maps within the App. Fetches information from a SQL database, and allows for the editing and deleting entries from within the app.

Project 2: Interactable Spreadsheet

This project involves two programs created using a Java library called Processing

The first project is a 2D interactable spreadsheet, where you can insert an excel spreadsheet which contains Name, X, Group, Year of Birth, Grade, Gender, and Y. This is then displayed in a 2D Graph, where the characteristics of each object is shown, grade indicates size, group indicates shape, and its displayed on the correct coordinates. You can interact with this object, change its characteristics and move it on either X and Y and save it back to the file, with the updated parameters.


The second project is a 3D interactable spreadsheet, it includes an additional parameter Z and is used within a 3D space, it has additional features where you can move multiple objects at once, delete objects, undo actions, copy objects, add different characteristics to objects such as wireframe, colour, outline, and when saved its added to the output of the file. The parameters for characteristics will then be applied when the document is added back.

Project 3: Music playlist database

This project uses JavaScript, SQL and PHP to create a music playlist database with customised accounts

The project was deployed on a live server within the university site. The project functioned similar to spotify, where users can search for music within the web application and play it. Users can also create personalised accounts where they can create their own playlist and add songs directly to it. The type of account the user had dictated the amount of features available, with "Free" only having the ability to create 3 personalised accounts, and "Premium" creating infinite.

Project 4: Visualisation of Large Relational Datasets and Multidimensional Datasets

Visual graphs created Tree-map, DOITree and EncCon Graph Techniques

Figures 1.1 and 1.2, depicted below, illustrates a tree map created using Tableau and was sourced from Kiezun (2023) containing 3.3k nodes. The treemap represents all movies with over 50,000 ratings released from 1970 to 2023 and was filtered to show only the movies grossed over 1 billion dollars USD. In figure 1.2 an additional filter is applied to exclude any ratings below 7.5, and the colour scheme was changed. This shows how easy it is for viewers to adjust the tree-map and change attributes in real-time (Johnson and Shneiderman, 1991). In both figures the average ratings is mapped to colour, with the darker shade of the colour indicating the higher rating. Whilst the size of the rectangle is mapped to the total gross of the movie. Insights that can be obtained from figure 1.1 are that “The Dark Knight Rises” and “The Lord of the Rings: The Return of the King” are two of the highest rated movies, as indicated by both having the darkest shade of blue among all the movies. However, the highest-grossing movies, “Avatar” and “Avengers: Endgame,” are not rated the highest. This shows us that although a movie is highly rated, it doesn’t necessarily correlate with the gross of that movie. This is further reinforced by the movie “Transformers: Age of Extinction,” which has a higher gross than “The Dark Knight Rises” despite being rated the lowest. Moreover, in figure 1.2 we can see that “Avatar: The Way of Water” is an outlier, as it’s surrounded by movies that are significantly higher rated than it.

Project 5: Retro in Disguise

A 3D Game created using C# within Unity

Retro in Disguise was my Year 12 project, which I received first place and a major works award for. The project was created using C# in Unity, and used various assets and animations from the Unity workshop. Players were tasked with traversing rooms to escape, using both colour-changing and size manipulation to navigate obstacles. Changing size altered the speed and jump height of the character, and changing colour allowed the character to traverse different rooms.
Throughout the game, strategically placed in-game hints aid players, presenting detailed images illustrating how button presses would affect the character's abilities. These hints were presented to the user in a linear progression to facilitate their understanding of the game mechanics.
